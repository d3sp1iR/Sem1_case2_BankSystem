## Описание проекта
Консольное приложение на C++, эмулирующее простую банковскую систему. Позволяет создавать счета, проводить операции пополнения и снятия, а также вычислять общий баланс всех счетов банка.

### Класс `BankAccount`
*   **Поля:** `accountNumber`, `ownerName`, `balance`.
*   **Методы:** `deposit()`, `withdraw()`, `getBalance()`, `printStatement()`, `getAccountNumber()`, `getOwnerName()`.

### Класс `Bank`
*   **Поля:** `std::vector<BankAccount> accounts`.
*   **Методы:** `createAccount()`, `getAccount()`, `totalBankBalance()`, `listAllAccounts()`.

## Инструкция по сборке и запуску
Проект использует `CMake` и предназначен для сборки в среде `WSL2` с компилятором `g++`.

1.  **Клонирование репозитория:**
    ```bash
    git clone <URL-вашего-репозитория>
    cd project-bank-system
    ```
2.  **Конфигурация CMake:**
    Откройте папку в VS Code (с расширением CMake Tools) и выберите компилятор (Kit), либо выполните в терминале:
    ```bash
    cmake -B build
    ```
3.  **Сборка проекта:**
    ```bash
    cmake --build build
    ```
    Или нажмите `F7` в VS Code.
4.  **Запуск программы:**
    ```bash
    ./build/BankSystem
    ```

## Проблемы и решения (Работа с нейросетью)
В ходе разработки мы использовали нейросеть для решения следующих задач:

1.  **Запрос:** "Как правильно организовать CMakeLists.txt для проекта с несколькими файлами .cpp и .h в папках include и src?"
    *   **Ответ нейросети:** (Краткая выдержка из ответа, например, про использование `include_directories` и перечисление всех файлов в `add_executable`).
2.  **Запрос:** "Как сгенерировать уникальные номера счетов в C++ для класса Bank?"
    *   **Ответ нейросети:** Предложила использовать статическую переменную или поле класса `nextAccountNumber` с инкрементом и преобразованием в строку через `std::ostringstream`. Это и было реализовано.
3.  **Запрос:** "В методе `listAllAccounts` я пытаюсь вывести имя владельца, но получаю ошибку компиляции. В чем проблема?"
    *   **Ответ нейросети:** Объяснила, что поле `ownerName` приватное, и для доступа к нему из класса `Bank` необходимо создать публичный метод-геттер (например, `getOwnerName()`) в классе `BankAccount`. Ошибка была исправлена.
